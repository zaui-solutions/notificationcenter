Platform Operations (System Health & Configuration)

Monitors and alerts on critical system configurations, data quality, and pending expirations that can interrupt sales or violate compliance. This is your engine health check, categorized by impact.

Critical: Sales Failure (Red)

Events that indicate a configuration has expired, immediately stopping bookings, breaking links, or causing operational outages.

  • Expired Product: {name} No Longer Available Product passed expiration date. Action required to restore availability.
  • Expired Departure Schedule: No Longer Bookable All departure times have passed. Activity is now unbookable.
  • Booking Engine {portalName} has expired Booking engine passed expiration date but is still live, causing broken links.

Warning: Proactive Expirations (Yellow)

Alerts that provide advance notice of scheduled or pending expirations that require action to prevent a future sales interruption.

  • Upcoming Expiration: Price Code / Rate Code Price code or account-level rate code is expiring soon, impacting pricing or B2B contracts.
  • Upcoming Expiration: Departure Times / Activity Departure times or the activity itself are set to expire soon, stopping future bookings.
  • Upcoming Expiration: Booking Engine / Seasonal Hours Booking portal or seasonal business hours are nearing expiration.
  • Upcoming Expiration: Pickup/Dropoff / Channel Rule Logistic locations, equipment downtime, or channel rules are set to expire.

Warning: Data Quality (Indigo)

Alerts focused on missing content that degrades product presentation and reduces conversion rates across all sales channels.

  • Missing Image: {name} has no assigned media Product is missing a required image, resulting in poor presentation.
  • Missing primary description: {name} has no description Product is missing its primary description, impacting SEO and customer information.
  • Empty Category: {name} Has No Assigned Items (Info) Category is empty, confusing customers or cluttering configuration.

Security & Access (Compliance & Threat Detection)

The essential tool for IT teams and compliance officers. Provides a detailed audit trail and real-time alerts on suspicious login behavior, user governance changes, and potential security threats. We categorize events by risk level for immediate action.

Low Risk: Account Hygiene (Green)

Automated deactivation of dormant accounts to reduce risk exposure and maintain performance.

  • Customer Inactivity Deactivation Account deactivated due to prolonged non-use.
  • Employee Auto-Deactivation (Inactivity) Employee account auto-deactivated based on retention thresholds.
  • Agent Auto-Deactivation (Inactivity) Agent account auto-deactivated based on retention thresholds.

Medium Risk: Anomalous Login (Yellow)

Events indicating potential unauthorized access attempts or unusual user behavior patterns.

  • Multiple Failed Logins Logged for Customer, Agent, and Employee accounts (Brute-force detection).
  • New Device/Location Login Login from a new, unrecognized device or location.
  • Concurrent Login Detected Same user logged in from two distant IPs simultaneously.

  • Login Outside Work Hours Employee access detected outside expected business hours.

High Risk: System Integrity (Red)

Critical events indicating potential attacks or sensitive modifications that affect data or system security.

  • Potential Code Injection Attempt Suspicious or malicious code detected in submitted content/config.
  • Employee Permissions Update Global menu/module permissions modified (High-value target).
  • Sensitive Admin Action Logged Action like clearing credit card data performed by admin-level user.

Announcements & Releases (Product Updates)

Ensures your sales and operations teams are instantly aware of ZAUI's continuous platform evolution. This acts as the centralized source of truth for new features and system updates.

Feature Rollout Notification

Immediate alerts when a major new capability is released (e.g., the new Passes system, enhanced Calendar Overview, or dynamic pricing tools).

Platform Release Notes

Standardized delivery of release notes and version updates, ensuring all internal stakeholders are working from the same product knowledge base.

Support Tickets (Service Communication)

Centralizes all open support issues with the ZAUI team (e.g., Zendesk), ensuring no critical issue or escalation is missed, providing a single source for status updates.

Real-Time Status Updates

Instant notification of ticket creation, status changes, and internal escalation to engineering or product teams.

Agent Response Notification

Alerts the relevant user immediately when a support agent has replied to an inquiry or requested more information.

Integrations & Marketplace (Connectivity Health)

Protects your external revenue pipelines. Monitors the health and stability of all third-party connections, including OTAs, payment gateways, and direct API links.

API Change Notification

Notifies technical users when there are updates, changes, or deprecations to the core ZAUI API, referencing the docs.zaui.com resource.

Channel Health Monitoring

Alerts to instability or failure in connections to key partners like Viator or Expedia, or payment systems (WePay/Alipay).
